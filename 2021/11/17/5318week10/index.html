COMP5318 Week 10

Density-based Methods

Partitioning(e.g.K-means) are designed to find spherical-shaped clusters. It have difficulty finding clusters of arbitrary shape, such as S shape and oval clusters.

Density-based Clustering Clustering based on density(a local cluster criterion), such as density-connected points.
Features and Characteristics of Density-based Clustering
· Discover clusters of arbitrary shape
· Handle noise
· One scan (only examine the local region to justify density)
· Need density parameters as termination condition

基于密度(一种局部聚类准则)的聚类，如密度连通点。

基于密度聚类的特征和特点

·发现任意形状的簇

·处理噪音

·一次扫描(仅检查局部区域以证明密度)

·终止条件需要密度参数

Density-based clustering locates regions of high density that are separated from one another by regions of low density.

基于密度的聚类定位密度高的区域，这些区域被密度低的区域分开。

DBSCAN

DBSCAN is a simple and effective density-based clustering algorithm that illustrates a number of concepts that are important for any density-based clustering approach.

DBSCAN是一种简单而有效的基于密度的聚类算法，它说明了许多对任何基于密度的聚类方法都很重要的概念。

A density-based notion of cluster
·A cluster Is defined as maximal set of density-connected points
·Density=number of points within a specified radius(Eps)

基于密度的集群概念

·聚类定义为密度连通点的最大集合

·密度=指定半径内的点数(Eps)

·A core point if it has at least a specified number of points(MinPts)within Eps
·These are points that are at the interior of a cluster
·Counts the point itself
·A border point is not a core point,but is in the neighborhood of a core point·A noise/outlier point is any point that is not a core point or a border point

如何选择半径和最小点数

Method for selecting DBSCAN parameters .
· Decide how many points you want ig a dense region: MinPts.
/Suppose we want core points to have at least k E-neighbors)
· Determine the distance from each point to its k-th nearest neighbor called the klist
· For points that belong to some cluster, the value of klist will be small lifk is not larger than the cluster sizel
· For points that are not in a cluster, such as noise points, the kdlist will be relatively large

选择DBSCAN参数的方法。

决定一个密集区域需要多少个点:MinPts。

/假设我们希望核心点至少有k个e邻居)

·确定每个点到第k个最近邻(klist)的距离

·对于属于某个集群的点，klist的值较小，lifk不大于集群大小

·对于不在集群中的点，比如噪声点，kdlist会比较大

· If we compute the kdist for all the data points for some k, sort them in increasing order, and then plot the sorted values, we expect to see a sharp change at the value of kdist that corresponds to a suitable value of E.
· If we select this dividing distance as the e parameter and take the value of k as the MinPts parameter, then points for which kdist is less than e will be labeled as core points, while other points will be labeled as noise or border points.

·如果我们计算某个k的所有数据点的kdist，对它们进行递增排序，然后绘制出排序后的值，我们期望在对应于合适的E值的kdist值处看到一个急剧的变化。

·选取该分割距离作为e参数，取k的值作为MinPts参数，则将kdist小于e的点标记为核心点，其他点标记为噪声点或边界点。

Determining EPS and MinPts

·ldea is that for points in a cluster,their kth nearest neighbors are at roughly the same distance
·Noise points have the kth nearest neighbor at farther distance
·Hence,plot sorted distance of every point to its kth nearest neighbor

想法是，对于一个星系团中的点，它们的第k近邻的距离大致相同

·噪声点在更远的距离有第k个最近邻

·因此，绘制每个点到其第k个最近邻的排序距离

DBSCAN can have trouble with density if the density of clusters varies widely.

In this toy data, the density of the clusters and noise regions is indicated by their darkness. The noise around the pair of denser clusters,A and B, has the same density as clusters C and D.

For a fixed MinPts, if the Eps threshold is chosen so that DBSCAN finds C and D as distinct clusters, with the points surrounding them as noise, then A and B and the points surrounding them will become a single cluster.
If the Eps threshold is set so that DBSCAN finds A and B as separate clusters, and the points surrounding them are marked as noise, then C,D, and the points surrounding them will also be marked as noise.

在这个玩具数据中，星团和噪声区域的密度是由它们的黑暗来表示的。 这对密度较大的星团A和B周围的噪声与星团C和D的密度相同。

对于一个固定的MinPts，如果选择Eps阈值，使DBSCAN发现C和D是不同的聚类，其周围的点是噪声，那么a和B及其周围的点将成为一个单独的聚类。

如果设置Eps阈值，使DBSCAN将A和B作为单独的聚类，并将其周围的点标记为噪声，则C、D及其周围的点也将标记为噪声。

Clustering Algorithms:Grid-based Clustering

Grid-based Clustering Method A grid is an efficient way to organize a set of data, at least in low dimensions.
Explore multi-resolution grid data structure in clustering
· Partition the data space into a finite number of cells to form a grid structure
· Find clusters(dense regions) from the cells in the grid structure

网格是组织一组数据的有效方法，至少在低维情况下是这样。

探讨聚类中的多分辨率网格数据结构

·将数据空间划分为有限数量的单元，形成网格结构

·从网格结构中的细胞中寻找簇(密集区域)

Features and Challenges of a typical grid-based algorithm

·Efficiency and scalability:# of cells <# of data points
·Uniformity:Uniform but hard to handle highly irregular data distributions
·Locality:Limited by pre-defined cell sizes,borders,and the density threshold
·Curse of dimensionality:Hard to cluster high-dimensional data

·效率和可扩展性:单元数量<数据点数量

·均匀性:均匀但难以处理高度不规则的数据分布

·局域性:受预定义的单元格大小、边界和密度阈值的限制

·维数诅咒:高维数据难以聚类

Good Clustering?

A good clustering method will produce high quality clusters which should have
· High intra-class similarity: Cohesive within clusters
· Low inter-class similarity: Distinctive between clusters Quality function
· Usually a separate”quality”function that measure the “goodness”of a cluster
· Difficult to define “similar enough”or “good enough”) quite subjective..
Similarity measure is critical for cluster analysis
· What similarity measure would you use?

Clustering Evaluation

Clustering Evaluation: Evaluating the goodness of clustering results
· No commonly recognized best suitable measure in practice Three categorization of measures: External, internal, and relative

  1. External: Supervised, employ criteria not inherent to the dataset Compare a clustering against prior or expert-specified knowledge
    (i.e., the ground truth) using certain clustering quality measure
  2. Internal: Unsupervised, criteria derived from data itself
    · Evaluate the goodness of a clustering by considering how well the clusters are separated and how compact the clusters are
    ·e.g, silhouette coefficient, calinski-harabasz Index, Davies-Bouldin Index
  3. Relative: Directly compare different clusterings, usually those obtained via different parameter settings for the same algorithm

聚类评价:评价聚类结果的优度

·实践中没有公认的最合适的措施三种措施:外部措施、内部措施和相对措施

\1. 将聚类与先前的或专家指定的知识进行比较

(即地面真实值)使用一定的聚类质量度量

\2. 内部:无监督的，从数据本身派生的标准

·通过考虑集群的分离情况和集群的紧凑程度来评估集群的优劣

·e。 g、剪影系数、calinski-harabasz指数、Davies-Bouldin指数

\3. 相对:直接比较不同的聚类，通常是同一算法通过不同参数设置得到的聚类
